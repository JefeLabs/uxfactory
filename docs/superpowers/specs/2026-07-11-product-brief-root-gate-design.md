# Product brief as the root gate for artifact seeding (2026-07-11)

## Problem

Nothing requires a product brief before seeding other artifacts. The enforced
dependency graph (`ARTIFACT_PREREQS`) has no `product-brief` edges; the panel's
Seed action is a one-click "derive-from-project draft (no interview)" that
never chains prerequisites; the worker seeds any artifact with "no prior
artifacts to honor"; and the brief has no validator. The brief only matters at
design-generation time (page-tier grounding chips).

The consequence is quiet corruption of the product's core promise. The brief is
the one artifact that cannot be derived, because it IS the intent — every other
artifact is a derivation from it. With no brief, a model does not fail loudly;
it invents a plausible product, and the fiction compounds through audience →
personas → stories → features. The gates then dutifully verify designs against
requirements the AI made up: "Generated by AI. Verified against your intent"
becomes "verified against invented intent."

## Decisions (user, 2026-07-11)

1. **Gate scope: ALL artifacts.** One rule, no exceptions — the brief is the
   root axiom. Even design-system choices should trace to positioning.
2. **Brief origin: user-authored, AI-structured.** The user answers the
   base-minimum questions in their own words; the producer may structure and
   format those answers into the brief but never invent content. Pasting or
   writing a brief directly in the artifact editor also counts.

## Design

### One new concept: the root artifact (`@uxfactory/spec`)

`product-brief` is declared the **root artifact** of the artifact graph:
user-authored, and a precondition for creating anything else.

- Export `ROOT_ARTIFACT = "product-brief"` plus a helper the panel and worker
  share, e.g. `requiresRootArtifact(artifactId): boolean` (true for every id
  except the root itself).
- This is deliberately NOT modeled through `ARTIFACT_PREREQS`. Prereq
  semantics mean "auto-chain an AI interview before this one," which is
  exactly what the brief must never be — and Seed ignores chains anyway.
  Prereqs order *derivations*; the root is the *axiom*. The asymmetry gets its
  own named concept.
- **"A brief exists" = the resolved artifact file exists and is non-empty**
  (registry-resolved path with the legacy-location fallback, same resolution
  reads use everywhere else). No section-parsing validator judging the user's
  prose: the intake path guarantees the base minimum for briefs created
  in-panel, and a pasted brief must count without fighting a format checker.

### The base minimum is already defined

`ARTIFACT_ELICITATION["product-brief"]` already holds four all-[E] (user-
answered, never derived) questions: problem/for-whom, measurable outcomes,
out-of-scope, non-negotiable constraints. The intake dialog renders exactly
these; all four are required before submission. No new question vocabulary.

### Panel enforcement (`@uxfactory/plugin`)

When no brief exists:

- Every non-brief artifact's **Seed and Create actions are disabled**, with a
  tooltip: "Supply your product brief first — every artifact derives from it."
  This covers ALL entry points that enqueue `generate-artifact` for a
  non-brief target: the Artifacts rows (Seed + Create), the ArtifactEditor's
  Regenerate, and the composer chips' create affordances.
- The Artifacts screen shows a **callout banner** ("Start with your product
  brief — everything else derives from it") whose CTA opens the brief intake.
- The **brief itself loses its Seed button permanently** (there is no
  legitimate derive-from-nothing path). Its creation paths are:
  1. the four-question intake (Create dialog) — the user's answers travel in
     the job payload; dialog copy states the producer structures your answers
     and never invents;
  2. writing/pasting directly in the artifact editor.

Once a brief exists, all gates lift; nothing else changes. Regenerating the
brief later re-opens the intake (pre-filled where possible), not a bare seed.

### Worker enforcement (fail-closed, `clients/uxfactory-worker`)

Defense in depth for CLI/agent-driven enqueues that bypass the panel:

- A `generate-artifact` job for any **non-brief** target settles as a setup
  failure with an actionable message ("no product brief found — supply one in
  the panel or at <path> before seeding other artifacts") when no brief file
  exists (same exists-and-non-empty test, shared resolution).
- A **brief** job whose payload carries no user answers is refused the same
  way rather than invented. When answers are present, the producer prompt
  gains a binding clause: structure and format the user's answers; do not add
  claims they did not make.

### Explicitly unchanged

- **Bridge stays a pure relay** — no policy in it.
- **Design generation** (`generate-design`) keeps its existing grounding-chip
  + ungoverned-escape-hatch semantics; this gate governs artifact seeding,
  where the trace graph gets its facts. (Page tiers already list the brief as
  required/recommended there.)
- `ARTIFACT_PREREQS`, `AUTHORING_ORDER`, and the chaining machinery.
- No hard block on `uxfactory batch` / gating — a project with designs but no
  brief still gates; the root gate is about creating intent artifacts.

## Rejected alternatives

- **Prereq edges** (brief in `ARTIFACT_PREREQS` of everything): wrong
  semantics (auto-chained AI interview), and Seed ignores chains.
- **Worker-only gate**: fail-closed but no affordance — users would discover
  the rule through failed jobs.
- **AI draft + mandatory approval state**: rejected by decision 2; a drafted
  brief is still invented intent, and an approval flag invites rubber-stamping.

## Migration / compatibility

Projects with an existing brief (e.g. uxfio-demo) are unaffected. Projects
without one see the banner and disabled actions until a brief is supplied.
Old panel + new worker: the worker's fail-closed check still guards. New panel
+ old worker: the panel gate alone holds for panel-driven flows (the worker
check is redundant there anyway).

## Testing

- **spec**: `requiresRootArtifact` truth table; root constant exported;
  elicitation entry unchanged (four [E] questions, all required).
- **plugin**: banner + disabled Seed/Create with tooltip when brief missing
  (and enabled when present); brief row never shows Seed; intake requires all
  four answers and stamps them into the enqueue payload; ArtifactEditor
  Regenerate gated for non-brief keys; gates lift reactively once the brief
  row turns present.
- **worker**: non-brief job without brief file → setup failure with the
  actionable message; brief job without answers → refused; brief job with
  answers → producer prompt contains the structure-only clause; non-brief job
  with brief present → proceeds.
