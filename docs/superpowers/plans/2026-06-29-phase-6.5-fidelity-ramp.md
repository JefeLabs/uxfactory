# Phase 6.5 — Multi-Dimensional Fidelity Ramp + Rubric + High-Fidelity Renderer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `uxfactory batch` request work at a **fidelity VECTOR** `{ coverage, editorial, visual, flow }` (each ordinal 0–2) — where named presets expand to a default vector, per-dimension overrides layer on top, and **each existing gate binds independently on its own dimension** (`vector[gate.dimension] ≥ gate.minLevel`) — gating the run, compiling the per-dimension rubric, enforcing per-dimension readiness, and selecting a renderer by the `visual` dimension.

**Architecture:** A new PURE module `packages/uxfactory-cli/src/batch/fidelity.ts` owns the `Fidelity` vector type, the `PRESETS` name→vector table, `parseFidelity`/`resolveFidelity`, the `GATE_DIMENSION`+`GATE_MIN_LEVEL` per-gate binding maps, the per-dimension required-input + declared manifests, and the deterministic `checkReadiness` precondition. `runBatch` gains a `fidelity: Fidelity` input and runs only gates where `vector[GATE_DIMENSION[id]] ≥ GATE_MIN_LEVEL[id]` (below-threshold → `not-owed`; declared-future dims → `declared`; the report carries the resolved vector + the rubric). `batchCmd` resolves the vector (registry preset/vector as base + per-dimension flags as overrides; unset → exit 2), enforces per-dimension readiness (a missing required input → exit 2 with a structured list), runs the vector-scoped pass, and renders previews via a renderer selected by `fidelity.visual` (resvg at 0, lazy Playwright at ≥1 with a resvg fallback + a `declared` note). The batch SKILL.md drives the agentic loop (set vector → satisfy readiness/generate-missing → gate against the per-dimension rubric → revise → promote a dimension). No in-engine LLM; the engine stays deterministic and self-contained.

**Tech Stack:** Node `>=20.10`, TS 6.0.3, ESM/NodeNext with `.js` import extensions and `verbatimModuleSyntax`. Extends `@uxfactory/cli` (reuses `@uxfactory/spec` `Spec`/`DesignSpec`, `specToSvg`, `svgToPng`, `BridgeClient`, `EXIT`, `IO`, the existing five gate functions). `playwright` is added as an OPTIONAL devDependency (lazy-imported; never required by the hermetic suite). Tests are Vitest (`.ts`) with per-test temp dirs + the in-process `startBridge`. The batch-loop skill is markdown vendored into the Claude Code plugin by the existing Node vendor step.

### The model — the three tables this phase implements VERBATIM

**Dimensions (v1) — each an ordinal 0–2:**

| Dimension | 0 | 1 | 2 |
| --- | --- | --- | --- |
| `coverage` | none | core (required stories/states) | full |
| `editorial` | placeholder | real copy | reviewed/voice |
| `visual` | greybox | tokenized | branded |
| `flow` | none | connected | all-states |

(`a11y`, `seo` are future dimensions — declared, not yet checkable.)

**Presets — each expands to a default vector:**

| Preset | coverage | editorial | visual | flow |
| --- | --- | --- | --- | --- |
| `wireframe` | 1 | 0 | 0 | 0 |
| `content` | 1 | 1 | 0 | 0 |
| `visual` | 1 | 1 | 1 | 0 |
| `interactive` | 1 | 1 | 1 | 1 |
| `production` | 2 | 2 | 2 | 2 |

**Gate → dimension binding (a gate binds iff `vector[dimension] ≥ minLevel`):**

| Gate (existing id) | Dimension | minLevel | Required input (mandatory) |
| --- | --- | --- | --- |
| `requirement-coverage` | coverage | 1 | `stories` |
| `reuse` | coverage | 1 | — (optional; skip-and-declare) |
| `coverage-orphans` (advisory) | coverage | 1 | — |
| `token-conformance` | visual | 1 | `tokens` |
| `flow-reachability` | flow | 1 | `flow` |

**Required inputs by dimension level** (mandatory to *request* a batch): `coverage ≥ 1 → stories`; `visual ≥ 1 → tokens`; `flow ≥ 1 → flow`. `reuse` is never required. `editorial ≥ 1` and future dims (`a11y`/`seo`) are **declared, not blocking**.

**Resolution:** the base vector is the `--fidelity <preset>` flag if present, else the registry `fidelity` (a preset name OR a partial vector → full vector with missing dims = 0); per-dimension flags (`--coverage`/`--editorial`/`--visual`/`--flow <0-2>`) override individual dimensions on top. So `--fidelity wireframe --flow 1` → `{coverage:1,editorial:0,visual:0,flow:1}` (`flow-reachability` binds; `token-conformance` does not).

```ts
/**
 * The fidelity VECTOR (§6.5) — fidelity is `{ coverage, editorial, visual, flow }`, each an
 * ordinal 0–2, NOT a single scalar level. Named presets expand to a default vector;
 * per-dimension overrides layer on top. Each gate binds INDEPENDENTLY on its own dimension:
 * a gate binds iff `vector[GATE_DIMENSION[id]] ≥ GATE_MIN_LEVEL[id]`. Pure + deterministic:
 * no I/O, no clock, no LLM.
 */

/** The four checkable dimensions (a11y/seo are future, declared-only). */
export type Dimension = "coverage" | "editorial" | "visual" | "flow";

/** A full fidelity vector — each dimension an ordinal 0–2. */
export interface Fidelity {
  coverage: number;
  editorial: number;
  visual: number;
  flow: number;
}

/** The named presets, each expanding to a default vector. */
export type PresetName = "wireframe" | "content" | "visual" | "interactive" | "production";

/** The dimensions in canonical order. */
export const DIMENSIONS: Dimension[] = ["coverage", "editorial", "visual", "flow"];

/** The preset names in ramp order. */
export const PRESET_NAMES: PresetName[] = [
  "wireframe",
  "content",
  "visual",
  "interactive",
  "production",
];

/** Each preset's default vector (the §6.5 table, verbatim). */
export const PRESETS: Record<PresetName, Fidelity> = {
  wireframe: { coverage: 1, editorial: 0, visual: 0, flow: 0 },
  content: { coverage: 1, editorial: 1, visual: 0, flow: 0 },
  visual: { coverage: 1, editorial: 1, visual: 1, flow: 0 },
  interactive: { coverage: 1, editorial: 1, visual: 1, flow: 1 },
  production: { coverage: 2, editorial: 2, visual: 2, flow: 2 },
};

/** A single dimension level is an integer 0–2. */
function isLevel(n: unknown): n is number {
  return typeof n === "number" && Number.isInteger(n) && n >= 0 && n <= 2;
}

/** Parse one dimension level ("0".."2" or 0..2) into 0–2, or null. */
export function parseLevel(s: string | number): number | null {
  const n = typeof s === "number" ? s : Number(String(s).trim());
  if (typeof s === "string" && s.trim() === "") return null;
  return isLevel(n) ? n : null;
}

/**
 * Parse a fidelity INPUT into a full vector, or null.
 * - a string is a PRESET NAME (case-insensitive) → that preset's vector;
 * - an object is a PARTIAL VECTOR → a full vector with missing dims = 0.
 * Unknown presets, unknown dimensions, and out-of-range / non-integer levels are rejected.
 */
export function parseFidelity(input: string | Record<string, unknown>): Fidelity | null {
  if (typeof input === "string") {
    const key = input.trim().toLowerCase();
    if (Object.prototype.hasOwnProperty.call(PRESETS, key)) {
      return { ...PRESETS[key as PresetName] };
    }
    return null;
  }
  if (typeof input === "object" && input !== null && !Array.isArray(input)) {
    const out: Fidelity = { coverage: 0, editorial: 0, visual: 0, flow: 0 };
    for (const key of Object.keys(input)) {
      if (!(DIMENSIONS as string[]).includes(key)) return null; // unknown dimension → reject
      const v = input[key];
      if (!isLevel(v)) return null;
      out[key as Dimension] = v;
    }
    return out;
  }
  return null;
}

/** Apply per-dimension overrides on top of a base vector (returns a fresh vector). */
export function resolveFidelity(base: Fidelity, overrides: Partial<Fidelity>): Fidelity {
  const out: Fidelity = { ...base };
  for (const dim of DIMENSIONS) {
    const v = overrides[dim];
    if (v !== undefined) out[dim] = v;
  }
  return out;
}

/**
 * The dimension each EXISTING gate owes on. Keys are the actual gate ids from `checks.ts`.
 * `coverage-orphans` (advisory companion of `requirement-coverage`, also driven by `stories`)
 * owes on `coverage` so the coverage trio binds together.
 */
export const GATE_DIMENSION: Record<string, Dimension> = {
  "requirement-coverage": "coverage",
  reuse: "coverage",
  "coverage-orphans": "coverage",
  "token-conformance": "visual",
  "flow-reachability": "flow",
};

/** The minimum level on its dimension at which each gate becomes BINDING. */
export const GATE_MIN_LEVEL: Record<string, number> = {
  "requirement-coverage": 1,
  reuse: 1,
  "coverage-orphans": 1,
  "token-conformance": 1,
  "flow-reachability": 1,
};

/** The RUBRIC for a vector: every gate id that binds (`vector[dimension] ≥ minLevel`). */
export function bindingGateIds(v: Fidelity): string[] {
  return Object.keys(GATE_DIMENSION).filter((id) => {
    const dim = GATE_DIMENSION[id];
    const min = GATE_MIN_LEVEL[id];
    return dim !== undefined && min !== undefined && v[dim] >= min;
  });
}

/**
 * The MANDATORY registered inputs to REQUEST a batch at a vector: `stories` at coverage≥1,
 * `tokens` at visual≥1, `flow` at flow≥1. `reuse` is OPTIONAL (skip-and-declare) and never
 * appears here. `editorial` and future dims require no checkable input (declared only).
 */
export function requiredInputs(v: Fidelity): string[] {
  const out: string[] = [];
  if (v.coverage >= 1) out.push("stories");
  if (v.visual >= 1) out.push("tokens");
  if (v.flow >= 1) out.push("flow");
  return out;
}

/** A dimension that is required by the vector but has no checkable gate yet. */
export interface DeclaredFuture {
  dimension: string;
  level: number;
  reason: string;
}

/**
 * The DECLARED tiers for a vector — honest "required, not yet checked": `editorial` when
 * editorial≥1 (real copy is owed, no editorial gate yet), plus the future, not-yet-checkable
 * dimensions `a11y` and `seo`. Never blocking, never a silent pass.
 */
export function declaredFuture(v: Fidelity): DeclaredFuture[] {
  const out: DeclaredFuture[] = [];
  if (v.editorial >= 1) {
    out.push({
      dimension: "editorial",
      level: v.editorial,
      reason: "real copy required, no editorial gate yet",
    });
  }
  out.push({ dimension: "a11y", level: 0, reason: "future dimension, not yet checkable" });
  out.push({ dimension: "seo", level: 0, reason: "future dimension, not yet checkable" });
  return out;
}

/** A missing required input the agent must provide or generate before a batch can run. */
export interface ReadinessMissing {
  artifact: string;
  dimension: string;
  level: number;
  action: "provide-or-generate";
}

/** The readiness precondition result: ready + what's missing + what's declared-not-yet-checked. */
export interface ReadinessReport {
  ready: boolean;
  missing: ReadinessMissing[];
  declared: DeclaredFuture[];
}

/**
 * The PRECONDITION that gates whether a batch can be REQUESTED at a vector: are all REQUIRED
 * inputs (+ specs) present for the vector's dimensions? Missing any → not ready, each listed
 * with the dimension + level that requires it and the "provide-or-generate" action the
 * SKILL.md drives. The engine REPORTS what's missing; the skill generates it.
 */
export function checkReadiness(
  v: Fidelity,
  present: { stories: boolean; tokens: boolean; flow: boolean; specs: boolean },
): ReadinessReport {
  const missing: ReadinessMissing[] = [];
  if (!present.specs) {
    missing.push({
      artifact: "specs",

import {
  tokenConformance,
  requirementCoverage,
  coverageOrphans,
  reuse,
  flowReachability,
} from "./checks.js";
import type { CheckResult, Severity, LoadedSpec, TokenSet, StorySet, Flow } from "./checks.js";
import { GATE_DIMENSION, GATE_MIN_LEVEL, bindingGateIds, declaredFuture } from "./fidelity.js";
import type { Fidelity } from "./fidelity.js";
import type { Spec } from "@uxfactory/spec";

/** Everything a single deterministic batch pass needs (inputs already loaded; null = absent). */
export interface RunBatchInput {
  specs: LoadedSpec[];
  tokens: TokenSet | null;
  stories: StorySet | null;
  reuseSpecs: Spec[] | null;
  flow: Flow | null;
  /** The resolved fidelity vector — selects the binding gates (the rubric) per dimension. */
  fidelity: Fidelity;
}

/** The result of one deterministic, vector-scoped pass. */
export interface BatchReport {
  /** The resolved vector this pass ran at. */
  fidelity: Fidelity;
  /** The binding gate ids for this vector (the RUBRIC the loop judges against). */
  rubric: string[];
  checks: CheckResult[];
  mustPassFailed: boolean;
  clean: boolean;
}

/** One gate: its id, severity (for the not-owed entry), and how to run it. */
interface GateDef {
  id: string;
  severity: Severity;
  run: (input: RunBatchInput) => CheckResult;
}

/** The existing gates, in deterministic report order (ids match GATE_DIMENSION). */
const GATES: GateDef[] = [
  {
    id: "requirement-coverage",
    severity: "must",
    run: (i) => requirementCoverage(i.specs, i.stories),
  },
  { id: "coverage-orphans", severity: "advisory", run: (i) => coverageOrphans(i.specs, i.stories) },
  { id: "reuse", severity: "must", run: (i) => reuse(i.specs, i.reuseSpecs) },
  { id: "token-conformance", severity: "must", run: (i) => tokenConformance(i.specs, i.tokens) },
  { id: "flow-reachability", severity: "advisory", run: (i) => flowReachability(i.specs, i.flow) },
];

/**
 * Run the gates that BIND for `input.fidelity` (a gate binds iff
 * `vector[GATE_DIMENSION[id]] ≥ GATE_MIN_LEVEL[id]`) ONCE and aggregate. Gates below their
 * dimension threshold are reported `not-owed` (never run, never gate); declared dimensions
 * (editorial / future a11y/seo) are reported `declared` (advisory). FULLY DETERMINISTIC: no
 * async, no clock, no judge/LLM. `mustPassFailed` is true iff a BINDING `severity:"must"` gate
 * is `"fail"`; `clean = !mustPassFailed`. This single pass IS the loop-termination signal —
 * the SKILL.md loop, not the engine, iterates.
 */
export function runBatch(input: RunBatchInput): BatchReport {
  const { fidelity } = input;
  const checks: CheckResult[] = [];

  for (const gate of GATES) {
    const dim = GATE_DIMENSION[gate.id];
    const min = GATE_MIN_LEVEL[gate.id];
    if (dim !== undefined && min !== undefined && fidelity[dim] >= min) {
      checks.push(gate.run(input));
    } else {
      checks.push({
        id: gate.id,
        status: "not-owed",
        severity: gate.severity,
        findings: [],
        reason:
          dim !== undefined && min !== undefined
            ? `binds at ${dim} ≥ ${min}; this run has ${dim} = ${fidelity[dim]}`
            : "no dimension binding",
      });
    }
  }

  for (const d of declaredFuture(fidelity)) {
    checks.push({
      id: d.dimension,
      status: "declared",
      severity: "advisory",
      findings: [],
      reason: d.reason,
    });
  }

  const mustPassFailed = checks.some((c) => c.severity === "must" && c.status === "fail");
  return {
    fidelity,
    rubric: bindingGateIds(fidelity),
    checks,
    mustPassFailed,
    clean: !mustPassFailed,
  };
}

Replace `packages/uxfactory-cli/src/commands/batch.ts` with:

```ts
import { readFile, readdir, writeFile, mkdir } from "node:fs/promises";
import path from "node:path";
import { EXIT, TransportError } from "../exit.js";
import { loadSpec, printSpecProblem } from "../spec-file.js";
import { readRegistry } from "../batch/registry.js";
import { runBatch } from "../batch/run.js";
import { parseFidelity, parseLevel, resolveFidelity, checkReadiness, DIMENSIONS } from "../batch/fidelity.js";
import type { Fidelity, Dimension } from "../batch/fidelity.js";
import { specToSvg } from "../render/svg.js";
import type { LoadedSpec, TokenSet, StorySet, Flow, CheckResult } from "../batch/checks.js";
import type { BatchReport } from "../batch/run.js";
import type { Spec } from "@uxfactory/spec";
import type { IO } from "../io.js";
import type { BridgeClient } from "../client.js";

/** Flags for `uxfactory batch`. */
export interface BatchFlags {
  json?: boolean;
  stage?: boolean;
  dataDir: string;
  /** Repo root where uxfactory.batch.json + the design/ inputs live (default process.cwd()). */
  cwd?: string;
  /** Override the registry's base preset (a preset name). */
  fidelity?: string;
  /** Per-dimension overrides (each "0".."2"). */
  coverage?: string;
  editorial?: string;
  visual?: string;
  flow?: string;
}

/** Read + JSON-parse a registered input; throws on any failure (→ setup error). */
async function readJson<T>(file: string): Promise<T> {
  return JSON.parse(await readFile(file, "utf8")) as T;
}

/**
 * `uxfactory batch <dir>` — ONE deterministic, self-contained offline pass at a chosen fidelity
 * VECTOR (§6.5/§13). Resolves the vector (base = --fidelity preset, else registry fidelity;
 * per-dimension flags override; unset base → 2), enforces the per-dimension readiness
 * precondition (a missing required input → 2 with a structured list), runs only the gates that
 * bind per dimension (below threshold → not-owed; declared dims → declared), writes
 * vector-rendered previews + a report under `.uxfactory/batch/`, optionally stages a clean
 * batch, and returns the loop-termination exit code: 0 clean / 1 must-pass failed / 2 setup.
 */
export async function batchCmd(
  specsDir: string,
  flags: BatchFlags,
  io: IO,
  client: BridgeClient,
): Promise<number> {
  const cwd = flags.cwd ?? process.cwd();

  // 1. registry (absent/invalid → 2)
  const reg = await readRegistry(path.join(cwd, "uxfactory.batch.json"));
  if (!reg.ok) {
    io.err(reg.message);
    return EXIT.TRANSPORT;
  }

  // 2. resolve the vector: base = --fidelity preset, else registry fidelity. Unset → setup error.
  const baseInput = flags.fidelity ?? reg.registry.fidelity;
  if (baseInput === undefined || baseInput === null) {
    io.err("set a fidelity before requesting a batch.");
    return EXIT.TRANSPORT;
  }
  const base = parseFidelity(baseInput as string | Record<string, unknown>);
  if (base === null) {
    io.err(
      `invalid fidelity ${JSON.stringify(baseInput)} (use a preset wireframe|content|visual|interactive|production or a partial vector)`,
    );
    return EXIT.TRANSPORT;
  }
  // per-dimension flag overrides
  const flagDims: Record<Dimension, string | undefined> = {
    coverage: flags.coverage,
    editorial: flags.editorial,
    visual: flags.visual,
    flow: flags.flow,
  };
  const overrides: Partial<Fidelity> = {};
  for (const dim of DIMENSIONS) {
    const raw = flagDims[dim];
    if (raw !== undefined) {
      const n = parseLevel(raw);
      if (n === null) {
        io.err(`invalid --${dim} ${JSON.stringify(raw)} (use a level 0-2)`);
        return EXIT.TRANSPORT;
      }
      overrides[dim] = n;
    }
  }
  const fidelity = resolveFidelity(base, overrides);

  // 3. load + validate the batch specs (invalid/unreadable → 2)
  let entries: string[];
  try {
    entries = (await readdir(specsDir)).filter((f) => f.endsWith(".uxfactory.json")).sort();
  } catch {
    io.err(`cannot read specs directory ${specsDir}`);
    return EXIT.TRANSPORT;
  }
  if (entries.length === 0) {
    io.err(`no *.uxfactory.json specs found in ${specsDir}`);
    return EXIT.TRANSPORT;
  }
  const specs: LoadedSpec[] = [];
  for (const name of entries) {
    const full = path.join(specsDir, name);
    const result = await loadSpec(full);
    if (!result.ok) return printSpecProblem(io, result, flags.json);
    specs.push({ file: name, spec: result.spec as Spec });
  }

  // 4. readiness precondition — a missing REQUIRED input gates the request (→ 2).
  const present = {
    stories: reg.inputs.stories !== null,
    tokens: reg.inputs.tokens !== null,
    flow: reg.inputs.flow !== null,
    specs: specs.length > 0,
  };
  const readiness = checkReadiness(fidelity, present);
  if (!readiness.ready) {
    if (flags.json === true) {
      io.out(
        JSON.stringify({
          ready: false,
          fidelity,
          missing: readiness.missing,
          declared: readiness.declared,
        }),
      );
    } else {
      io.err(`batch not ready at ${JSON.stringify(fidelity)} — provide or generate the required input(s):`);
      for (const m of readiness.missing) {
        io.err(`  - ${m.artifact} (required at ${m.dimension} ≥ ${m.level}) — ${m.action}`);
      }
    }
    return EXIT.TRANSPORT;
  }

  // 5. load the registered inputs that EXIST (absent → null = skip; registered-but-unreadable → 2)
  let tokens: TokenSet | null = null;
  let stories: StorySet | null = null;
  let flow: Flow | null = null;
  let reuseSpecs: Spec[] | null = null;
  try {
    if (reg.inputs.tokens !== null) {
      tokens = await readJson<TokenSet>(reg.inputs.tokens);
      if (
        tokens.colors === null ||
        typeof tokens.colors !== "object" ||
        Array.isArray(tokens.colors)
      ) {
        io.err(
          `malformed tokens file: "colors" must be an object (got ${JSON.stringify(typeof tokens.colors)})`,
        );
        return EXIT.TRANSPORT;
      }
    }
    if (reg.inputs.stories !== null) {
      stories = await readJson<StorySet>(reg.inputs.stories);
      if (!Array.isArray(stories.stories)) {
        io.err(
          `malformed stories file: "stories" must be an array (got ${JSON.stringify(typeof stories.stories)})`,
        );
        return EXIT.TRANSPORT;
      }
    }
    if (reg.inputs.flow !== null) flow = await readJson<Flow>(reg.inputs.flow);
    if (reg.inputs.reuse.length > 0) {
      reuseSpecs = [];
      for (const file of reg.inputs.reuse) {
        const result = await loadSpec(file);
        if (!result.ok) {
          io.err(`unreadable/invalid reuse spec: ${file}`);
          return EXIT.TRANSPORT;
        }
        reuseSpecs.push(result.spec as Spec);
      }
    }
  } catch (err) {
    io.err(`cannot read a registered input: ${(err as Error).message}`);
    return EXIT.TRANSPORT;
  }

  // 6. ONE deterministic, vector-scoped pass
  const report: BatchReport = runBatch({ specs, tokens, stories, reuseSpecs, flow, fidelity });

  // 7. offline previews per spec — SVG source + a raster selected by the `visual` dimension.
  //    The selector is lazy-imported so resvg/playwright are never touched on other commands.
  const batchDir = path.join(flags.dataDir, "batch");
  const previewDir = path.join(batchDir, "previews");
  await mkdir(previewDir, { recursive: true });
  const previews = new Map<string, string>();
  const { selectRasterizer } = await import("../render/raster-select.js");
  const rasterize = selectRasterizer(fidelity.visual);
  let rendererDowngrade: string | null = null;
  for (const s of specs) {
    const svg = specToSvg(s.spec);
    previews.set(s.file, svg);
    const baseName = s.file.replace(/\.[^.]+$/, "");
    await writeFile(path.join(previewDir, baseName + ".svg"), svg, "utf8");
    const raster = await rasterize(svg);
    await writeFile(path.join(previewDir, baseName + ".png"), raster.png);
    if (raster.downgraded && rendererDowngrade === null) {
      rendererDowngrade = raster.note ?? "high-fidelity renderer unavailable, used resvg";
    }
  }
  if (rendererDowngrade !== null) {
    const declared: CheckResult = {
      id: "high-fidelity-renderer",
      status: "declared",
      severity: "advisory",
      findings: [],
      reason: rendererDowngrade,
    };
    report.checks.push(declared);
  }

  // 8. report.json + summary (carries the vector + rubric + not-owed + declared)
  const reportDoc = { specs: specs.map((s) => s.file), ...report };
  await writeFile(path.join(batchDir, "report.json"), JSON.stringify(reportDoc, null, 2), "utf8");
  if (flags.json === true) {
    io.out(JSON.stringify(reportDoc));
  } else {
    io.out(
      `batch: ${report.clean ? "clean" : "FAILED"} — ${specs.length} spec(s) at ${JSON.stringify(report.fidelity)}`,
    );
    io.out(`  rubric: ${report.rubric.join(", ")}`);
    for (const c of report.checks) {
      const tag = `[${c.severity}] ${c.id}: ${c.status}`;
      io.out(c.reason !== undefined ? `  ${tag} (${c.reason})` : `  ${tag}`);
      for (const f of c.findings) io.out(`    - ${f.detail}`);
    }
  }

  // 9. stage a clean batch to the bridge (bridge error → 2)
  if (flags.stage === true && report.clean) {
    try {
      const { batchId } = await client.postBatch(
        specs.map((s) => ({ spec: s.spec, preview: previews.get(s.file) })),
      );
      io.out(`staged batch ${batchId} for approval`);
    } catch (err) {
      if (err instanceof TransportError) {
        io.err(err.message);
        return EXIT.TRANSPORT;
      }
      throw err;
    }
  }

  // 10. loop-termination exit code
  return report.mustPassFailed ? EXIT.GATE_FAIL : EXIT.OK;
}

```ts
/**
 * The ONLY module that imports `playwright` — an OPTIONAL, heavy dependency (browser
 * binaries). It is LAZY-imported by the renderer selector at `visual ≥ 1`, so the dep is
 * never loaded at visual 0 or on unrelated CLI commands. Renders the preview SVG inside a
 * headless Chromium page and screenshots it to a PNG.
 *
 * THROWS when `playwright` or its browser is unavailable (import error / launch error), so
 * the selector (`selectRasterizer`) can fall back to resvg and DECLARE the downgrade — a
 * missing high-fidelity renderer is never a hard batch failure.
 */
export async function svgToPngPlaywright(svg: string): Promise<Buffer> {
  const { chromium } = await import("playwright");
  const browser = await chromium.launch({ headless: true });
  try {
    const page = await browser.newPage();
    const html =
      `<!doctype html><html><head><meta charset="utf-8">` +
      `<style>html,body{margin:0;padding:0;background:#fff}</style></head>` +
      `<body>${svg}</body></html>`;
    await page.setContent(html, { waitUntil: "load" });
    const el = await page.$("svg");
    const buf =
      el !== null ? await el.screenshot({ type: "png" }) : await page.screenshot({ type: "png" });
    return buf;
  } finally {
    await browser.close();
  }
}

import { svgToPng } from "./raster.js";

/** A rasterized preview + which renderer produced it and whether it was downgraded. */
export interface RasterResult {
  png: Buffer;
  renderer: "resvg" | "playwright";
  downgraded: boolean;
  /** Set only on a downgrade — the declared limitation to surface in the report. */
  note?: string;
}

/** A function that rasterizes one preview SVG to a PNG (already bound to a visual level). */
export type Rasterizer = (svg: string) => Promise<RasterResult>;

/**
 * Select the preview rasterizer by the run's `visual` dimension (§6.5): resvg at `visual = 0`
 * (greybox), the high-fidelity Playwright renderer at `visual ≥ 1` (tokenized/branded). When
 * Playwright (or its browser) is unavailable at `visual ≥ 1`, the returned rasterizer FALLS
 * BACK to resvg and DECLARES the downgrade — never a hard error. Playwright is lazy-imported
 * (only at `visual ≥ 1`) so the optional heavy dep is never loaded at visual 0.
 */
export function selectRasterizer(visualLevel: number): Rasterizer {
  if (visualLevel < 1) {
    return async (svg: string) => ({ png: svgToPng(svg), renderer: "resvg", downgraded: false });
  }
  return async (svg: string) => {
    try {
      const { svgToPngPlaywright } = await import("./raster-playwright.js");
      return { png: await svgToPngPlaywright(svg), renderer: "playwright", downgraded: false };
    } catch (err) {
      return {
        png: svgToPng(svg),
        renderer: "resvg",
        downgraded: true,
        note: `high-fidelity renderer unavailable, used resvg (${err instanceof Error ? err.message : String(err)})`,
      };
    }
  };
}

# UXFactory — offline batch loop (multi-dimensional fidelity)

This skill teaches you to author **one or more UI components / a screen / a set of pages** entirely offline as UXFactory specs, **at a chosen fidelity vector**, and to drive them to a clean mechanical bar before a human ever looks. There is no judge and no scoring engine here: the CLI runs **one deterministic gate pass** scoped to the vector, and its **exit code** tells you whether to stop or revise. The subjective judgment — is the flow sensible, is the labeling clear — is **yours**, guided by the gate's findings. You are the loop; `uxfactory batch` is the gate.

## When to use this skill

Use it when the user wants to **generate a batch of UI** — components, a page, or several pages / a screen-flow — as UXFactory specs, offline, and have it mechanically checked at a fidelity vector before review. Lead with this when there is no Figma session and the goal is to assemble and self-check a set of specs.

Do **not** use it for: the online single-spec render→verify loop (use the main `uxfactory` skill), pixel-faithful sign-off (the previews here are approximate), or freeform UI with no structured spec.

## Fidelity is a VECTOR, not one level

Different concerns advance independently: final copy on a greybox; a complete flow with placeholder content; styled visuals with no flow. So fidelity is a **vector over four dimensions**, each an ordinal **0–2**, and **each gate owes on its own dimension** — `token-conformance` binds on `visual`, `flow-reachability` on `flow`, regardless of the other dimensions. A gate binds iff `vector[dimension] ≥ the gate's minLevel`.

| Dimension   | 0           | 1                             | 2                |
| ----------- | ----------- | ----------------------------- | ---------------- |
| `coverage`  | none        | core (required stories/states) | full             |
| `editorial` | placeholder | real copy                     | reviewed / voice |
| `visual`    | greybox     | tokenized                     | branded          |
| `flow`      | none        | connected                     | all-states       |

(`a11y` and `seo` are future dimensions — **declared**, not yet checkable.)

### Presets — set one name, get a sensible vector

Named presets expand to a default vector. Set one preset and override any single dimension to express uneven maturity.

| Preset        | coverage | editorial | visual | flow |
| ------------- | -------- | --------- | ------ | ---- |
| `wireframe`   | 1        | 0         | 0      | 0    |
| `content`     | 1        | 1         | 0      | 0    |
| `visual`      | 1        | 1         | 1      | 0    |
| `interactive` | 1        | 1         | 1      | 1    |
| `production`  | 2        | 2         | 2      | 2    |

**Resolution:** the **base** is `--fidelity <preset>` (or the registry `fidelity`), then per-dimension flags override single axes. So `--fidelity wireframe --flow 1` = a greybox **with a connected flow** → `{coverage:1, editorial:0, visual:0, flow:1}` → `flow-reachability` binds, `token-conformance` does not.

### Gate → dimension binding

| Gate (must/advisory)                | Dimension  | Binds at | Required input        |
| ----------------------------------- | ---------- | -------- | --------------------- |
| requirement & state coverage (must) | `coverage` | 1        | stories               |
| reuse (must)                        | `coverage` | 1        | — (optional)          |
| coverage orphans (advisory)         | `coverage` | 1        | —                     |
| token conformance (must)            | `visual`   | 1        | tokens                |
| flow reachability (advisory)        | `flow`     | 1        | flow                  |

**Required inputs to request a batch:** `coverage ≥ 1 → stories`; `visual ≥ 1 → tokens`; `flow ≥ 1 → flow`. **`reuse` is never required** (leave it unregistered and the gate skip-and-declares). `editorial ≥ 1` owes real copy but has no gate yet → **declared**, not blocking.

## Setting the vector

- In `uxfactory.batch.json`: `"fidelity": "visual"` (a preset name) **or** a partial vector `"fidelity": { "coverage": 1, "visual": 1 }` (missing dims = 0).
- On the CLI: `--fidelity <preset>` sets the base; `--coverage` / `--editorial` / `--visual` / `--flow <0-2>` override single dimensions. Flags win over the registry.
- If neither a `--fidelity` flag nor a registry `fidelity` is set, `uxfactory batch` exits `2`: "set a fidelity before requesting a batch."

## The inputs

Two committed, authored things drive the gate (you do not invent these — the user owns them):

- **`uxfactory.batch.json`** at the repo root — the registry. It carries the `fidelity` (preset name or partial vector), points at the guidance inputs, and carries an optional `maxIterations` budget:

```jsonc
{
  "version": 1,
  "fidelity": "visual", // a preset name, OR a partial vector like { "coverage": 1, "visual": 1 }
  "inputs": {
    "tokens": "design/tokens.ds.json", // name → hex color register (required at visual ≥ 1)
    "stories": "design/stories.json", // stories + acceptance criteria (required at coverage ≥ 1)
    "flow": "design/flow.json", // a declared step order (required at flow ≥ 1)
    "reuse": ["specs/existing.uxfactory.json"], // optional: specs to compose against, not duplicate
  },
  "maxIterations": 6,
}
```

- The **`design/`** folder it points at — the actual tokens, stories, and flow files. The specs you author live in their own directory (e.g. `specs/`), one `*.uxfactory.json` per component/screen/page.

### Minimal input shapes (v1)

- **tokens** (`tokens.ds.json`): `{ "colors": { "brand": "#1E88E5" } }`
- **stories** (`stories.json`): `{ "stories": [ { "id": "story-1", "role": "...", "goal": "...", "benefit": "...", "acceptanceCriteria": [ { "statement": "...", "impliedState": "empty|loading|error|success|edge" } ] } ] }`
- **flow** (`flow.json`): `{ "steps": ["<node-or-frame-name>", "..."] }` — an ordered sequence

## The loop

1. **Set a fidelity vector** — pick the preset closest to what you are producing (greybox → `wireframe`; hi-fi → `visual`; wired proto → `interactive`/`production`), then override any single dimension that is ahead or behind (e.g. `--fidelity content --flow 1`).
2. **Satisfy readiness.** Run the gate; on **exit `2` with a `missing` list**, the engine is telling you which required inputs are absent for the vector (each names the artifact, the dimension + level that requires it, and "provide or generate"). **Generate** the missing required inputs (author the `stories`/`tokens`/`flow` the vector needs and register them), then re-run. The engine reports what is missing; **you** generate it. Readiness is the precondition — a batch runs only when every required input for the vector exists.
3. **Author / revise the spec(s)** under a directory (e.g. `specs/`). Name things so the gate can trace them:
   - put each story's id in the **frame name** it satisfies (e.g. `story-1-home`),
   - put each acceptance-criterion **state keyword** in a node name (e.g. `home-empty-state`, `home-success-view`),
   - use **registered token colors** for every fill/stroke (binds when `visual ≥ 1`),
   - **reference** an existing spec's screen instead of redrawing it.
4. **Run the deterministic gate** — the gates that BIND for the vector are the **rubric**. Judge state against the rubric each iteration:
   ```bash
   uxfactory batch specs --fidelity visual   # one deterministic, vector-scoped pass
   ```
   It writes `.uxfactory/batch/report.json` (the vector, the rubric, every check) and `previews/<spec>.svg` + `previews/<spec>.png`.
5. **On exit `1`**, read the findings in `report.json` (uncovered stories/states, ad-hoc colors, duplicates) and **revise the spec(s)** to address each one — then re-run step 4.
6. **Stop** when `uxfactory batch` exits `0` (every binding must-pass gate is green) **or** you have spent `maxIterations` revisions. If the budget runs out with findings still open, surface the **best-effort** batch with the unmet findings listed — do **not** spin.
7. **Promote a dimension** (optional). Once the vector is clean, you may **promote** one dimension (e.g. raise `visual` from 0 to 1), re-run readiness (the higher dimension may need a new input — provide or generate it), and repeat the loop. Promotion is one resolved vector per call; you walk the dimensions.
8. **Hand to the human.** Once the vector is clean and the human approves, stage it:
   ```bash
   uxfactory batch specs --fidelity visual --stage   # posts the specs + previews for review/approval
   ```

The gate's **exit code is the termination condition** — you never decide "good enough" from a score; you decide from the binary gate plus your own read of the findings.

## The gates

`uxfactory batch` runs the gates that bind for the vector. Three are **must-pass** (they set the exit code); the rest are **advisory** (they never fail the batch):

- **requirement & state coverage** (must, binds at `coverage ≥ 1`) — every story id maps to ≥1 frame and every acceptance-criterion state maps to a node in that story's frame.
- **reuse** (must, binds at `coverage ≥ 1`) — a screen/component that already exists in a registered spec must be referenced, not regenerated.
- **token conformance** (must, binds at `visual ≥ 1`) — every fill/stroke must be a registered token color; ad-hoc values are findings.
- **flow reachability** (advisory, binds at `flow ≥ 1`) — if a flow declares a step order, each consecutive pair must be reachable along your connectors; unreachable pairs are advisory findings only.
- **coverage orphans** (advisory, binds at `coverage ≥ 1`) — frames whose names match no registered story id; advisory, never gates.

## Not-owed and declared

- A gate **below its dimension threshold** is reported `not-owed` — it is not part of this vector's rubric and never gates. (At `visual:0`, token conformance is `not-owed`.)
- A dimension that is owed but has no gate built yet (`editorial`, and the future `a11y`/`seo`) is reported `declared` — "required, not yet checked." Honest skip-and-declare, never a silent pass and never blocking.

## Skip-and-declare

A binding gate whose **input is not registered** is reported as `skipped` with a reason — never silently passed and never failed. "No flow registered" is honestly distinct from "reachability passed." If you need a check to run, register its input (and make sure the vector owes it).

## Exit codes — the loop-termination contract

| Code | Meaning                                                                                                       | What to do                                                        |
| ---- | ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------- |
| `0`  | Every binding must-pass gate is green                                                                         | **Stop the loop.** Hand off, or promote a dimension.            |
| `1`  | A binding must-pass gate failed                                                                               | Read `report.json` findings, revise the spec(s), re-run.        |
| `2`  | Setup: no fidelity set, NOT READY (a required input missing, with the list), bad/missing registry, unreadable input, --stage bridge error | Set fidelity / provide or generate the missing input / fix setup. |

## Outputs

Everything the pass produces is ephemeral and lives under **`.uxfactory/batch/`** (gitignored): `report.json` (the vector, the rubric, every check + findings) and `previews/<spec>.svg` + `previews/<spec>.png` (approximate offline previews; the raster renderer is selected by the `visual` dimension). The committed inputs (`uxfactory.batch.json`, `design/`) are never written to.

## Gotchas worth internalizing

- **The engine does not loop, score, or generate.** One call = one deterministic, vector-scoped pass. It reports what is missing; **you** generate it and iterate; the exit code stops you.
- **Set the vector first.** No fidelity = exit `2`. Pick the preset that fits, then override single dimensions.
- **Each gate owes on its own axis.** Raising `visual` lights up token conformance; raising `flow` lights up reachability — independently.
- **Readiness gates the request.** A required input missing for the vector = exit `2` with the list. Provide or generate it, then re-run.
- **Name for traceability.** Coverage and flow gates match on **names** — story ids in frame names, state keywords in node names, flow steps as node/frame names.
- **Previews are approximate** — good for review, not for pixel sign-off. The high-fidelity (Playwright) renderer at `visual ≥ 1` is optional; when absent the report declares the downgrade and falls back to the built-in raster.
- **`exit 2` is never a quality signal** — it means fidelity/readiness/registry/input/bridge; fix the setup, do not "revise the spec."
- **Don't spin.** Respect `maxIterations`; surface best-effort with the open findings when the budget is spent.
- **Promote deliberately.** A higher dimension may need a new input and binds more gates; re-run readiness when you raise one.

### Judgment calls (flag at review)

1. **Resolution: `--fidelity` is the base, per-dimension flags are the overrides.** `base = flags.fidelity ?? registry.fidelity` (a `--fidelity` preset replaces the registry base); the numeric per-dimension flags override single axes via `resolveFidelity`. This satisfies both "registry-as-base + flags-as-overrides" and the doc's `--fidelity wireframe --flow 1 → {1,0,0,1}` example. Alternative: registry always the base with `--fidelity` treated as a whole-vector override — rejected as it doesn't compose with per-dimension `resolveFidelity`.
2. **"Unset" means no base.** A vector requires a base (a `--fidelity` flag or registry `fidelity`); per-dimension flags alone do NOT establish a vector from all-zero (that would silently default the intent the design forbids). No base → exit 2.
3. **`parseFidelity` rejects unknown dimension keys.** A partial-vector object with a key outside `{coverage,editorial,visual,flow}` (e.g. `a11y`) is rejected — a11y/seo are future, declared-only dimensions, not part of the v1 checkable vector, so a typo or a premature dim is a validation error, not a silent pass.
4. **`declaredFuture` = editorial (when ≥1) + the future a11y/seo dims.** The vector model has no per-level artifact table; the honest "required, not yet checked" set is `editorial` (owed real copy, no gate) plus the always-declared future dimensions. `runBatch` surfaces each as a `declared` advisory check (`id` = the dimension name).
5. **`selectRasterizer(visualLevel)` returns a bound `Rasterizer`.** It selects by the `visual` dimension and returns `(svg) => Promise<RasterResult>`, so the batch loop binds once (`selectRasterizer(fidelity.visual)`) and rasterizes each spec — honoring the "select" name while staying practical.
